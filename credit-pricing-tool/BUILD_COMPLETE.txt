================================================================================
API OVERVIEW
================================================================================

All endpoints require Content-Type: application/json (except GET /health)

POST /api/rate
  Input: financials (dict), sector_id (str), business_description (optional)
  Output: sp_rating, moodys_rating, blended_rating, metrics
  Uses: Both S&P and Moody's engines with country_risk=2 (NZ)

POST /api/pricing/lookup
  Input: rating (str), tenor (int 1-5)
  Output: min_bps, max_bps, mid_bps
  Uses: Pricing matrix YAML lookup

POST /api/pricing/full-analysis
  Input: financials, sector_id, actual_rate_pct, facility_tenor, facility_type
  Output: ratings, spread, base_rate, expected_range, actual_rate, delta, interpretation
  Uses: Complete pipeline (rate → spread → base → expected → delta)

GET /api/base-rates
  Input: None
  Output: List of {bank, corporate_rate, working_capital_rate, last_updated}
  Uses: Hardcoded NZ bank defaults (ANZ, ASB, BNZ, Westpac, Kiwibank)

POST /api/extract/pdf
  Input: PDF file (multipart)
  Output: status, extracted_fields, filename, message
  Uses: Stub implementation (ready for OCR/NLP)

GET /health
  Input: None
  Output: {status: "healthy"}
  Uses: Simple health check

ARCHITECTURE
================================================================================

FastAPI Application:
api/main.py
├── FastAPI app creation
├── CORS middleware
├── Router mounting:
│   ├── api.rate.routes → /api/rate
│   ├── api.pricing.routes → /api/pricing/*
│   ├── api.scrape.routes → /api/base-rates
│   └── api.extract.routes → /api/extract/pdf
└── Health endpoint

Module Organization:
api/
├── main.py (application factory)
├── rate/
│   └── routes.py (rating endpoint logic)
├── pricing/
│   ├── engine.py (pricing computations)
│   └── routes.py (pricing endpoint logic)
├── scrape/
│   └── routes.py (base rates endpoint)
└── extract/
    └── routes.py (PDF extraction stub)

Data Flow:
Request → Pydantic validation → Engine computation → Response

Error Handling:
Input validation → Caught and returned as 400 Bad Request
Computation errors → Caught and returned as 500 Internal Server Error
Missing fields → Pydantic catches before handler
